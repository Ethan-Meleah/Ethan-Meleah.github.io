To begin talking about what the best HTML and CSS practices in regards to Classes and ID, we should first clarify what they are, what they do, and why we use them. Classes and ID are relatively simple concepts.

Any website you’ve ever seen is made with two core files. HTML and CSS. HTML is the file that determines what information is displayed, and how it is structured. CSS is how that information is visually displayed, changing the position, color, fonts etc. of the information.

HTML is ‘tag based’. For people who aren’t super technically knowledgeable, this means that content is separated into sections by something known as a ‘tag’.

But what if you want to only change one paragraph? Or only two? This is where classes and id’s come in.

An ID is a unique property. Unique in this case means that there can only be a single element that has this ID. Think of it like a person's driver's license. Every driver’s license can only correspond to a singular person. CSS can use an ID to make changes to a single element only.

If the CSS calls the element through it’s ID, the changes will only affect that element. Example below.

A Class is not a unique property. But it is used in a similar way to and ID. The only relevant difference between class and id is that class can be used for multiple different elements.

So when do we use them?

When you are choosing between using an ID or a Class, the absolute easiest way to choose which one to use is simple. Will you need to style another element in the same way as this one? If the answer isn’t a resounding no, you use a class, to be safe.

Best Practices

Now that we have the basics of information about CSS ID’s and Classes, we can talk about best practices for them. If you are unfamiliar with that term, we are going to talk about the best way that we can use them, for the most efficiency and clarity.

  • Don't use what you don't need. If you want to style every link on the page in the same way, don't use a link class, simply directly target the link element.
  • Use functional names. Use names that describe the function of the element. For example ‘navigation’ is better than ‘menu’, because it simply and cleanly describes what the element does.
  • Use names that are as short as possible, but as long as needed. Basically what it says. You want to make names just long enough to concisely convey what the function of element is.
